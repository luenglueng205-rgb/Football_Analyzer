交互范式对比：RLEF中枢下的 GWM 与 ZSA

在 Path C (以 RLEF 为中枢的三合一系统) 架构下，生成式世界模型 (GWM) 与 零样本套利 (ZSA) 的三种可能交互方式

模式一：GWM 作为沙盘，ZSA 作为提案者 (前置模拟验证)

机制：ZSA 发现套利机会（提案），但不直接执行。提案送入 GWM 进行未来状态推演。RLEF 根据 GWM 的模拟结果计算预期 Reward，决定是否在真实环境执行。

模式二：双轨并行，ZSA 作为异常捕获器 (长短线分离)

机制：RLEF 与 GWM 形成主内循环（慢思考，长线策略）。ZSA 作为“快车道”监听真实环境，发现极端机会立刻执行。ZSA 的真实盈亏作为强反馈注入 RLEF，强制 GWM 更新认知。

模式三：RLEF 作为动态路由中枢 (不确定性驱动)

机制：RLEF 评估当前环境的“熵”。稳态下，RLEF 依赖 GWM 推演决策；突变下（如突发红牌），RLEF 将控制权动态切换给 ZSA 进行高频套利。两者结果汇聚回 RLEF。
