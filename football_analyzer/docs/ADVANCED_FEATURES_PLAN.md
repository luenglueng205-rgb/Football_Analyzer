# 中国体育彩票足球智能分析系统 - 深度进阶计划 (Advanced AI-Native Plan)

## 核心设计哲学
**坚决不走传统臃肿的机器学习（ML）老路！**
我们将坚持目前的 **"LLM 大脑 (推理与编排) + Python 轻量级工具 (计算与执行)"** 的解耦架构。所有的深度分析能力都将通过新增原子工具（Atomic Skills）和优化多 Agent 辩论逻辑来实现，确保系统极度敏捷、可解释且易于维护。

---

## 阶段一：非结构化因素建模与蒙特卡洛模拟 (高性价比，见效最快)

### 1. 天气与裁判非结构化建模 (Weather & Referee Impact)
*   **目标**：解决目前系统对外部环境盲目的问题，提升“总进球”和“北京单场-上下单双”的预测准确率。
*   **实现方式**：
    *   新增工具 `get_match_environment(match_id)`：抓取比赛地实时天气（雨/雪/风速）和主裁判数据（场均出牌、判点球率）。
    *   **LLM 语义推理**：不需要复杂的特征工程。直接将数据喂给 `AnalystAgent`，利用大模型强大的常识推理能力。例如：“大雨导致场地湿滑，传控失误率上升，结合主裁安东尼·泰勒的严厉尺度，本场比赛犯规增加，节奏破碎。”
    *   **动作**：强制要求 LLM 在调用泊松分布工具前，根据环境因素动态下调双方的预期进球数 ($\mu$)。

### 2. 蒙特卡洛时间轴剧本模拟 (Monte Carlo Time-Series Simulation)
*   **目标**：打破静态 90 分钟概率的局限，针对“半全场”和“胜负过关”进行深度推演。
*   **实现方式**：
    *   新增 Python 工具 `simulate_match_timeline(home_xg, away_xg, iterations=10000)`。
    *   将 90 分钟切割为 90 个独立的时间片 (Time Bins)，基于泊松过程在毫秒级模拟 10,000 次比赛进程。
    *   统计诸如“上半场 0-0，下半场主队进球”的精确概率，直接映射到体彩“半全场”玩法的 9 个选项（胜胜、平胜、负负等）中，寻找高赔率漏洞。

## 阶段二：微观价值与聪明资金追踪 (触及核心，降维打击)

### 3. 微观球员价值量化 (Player-Level xG Impact)
*   **目标**：解决核心球员伤停对球队整体实力的精确衰减计算。
*   **实现方式**：
    *   引入外部轻量级数据源（如 FBref / API-Sports），获取球员的 **xG (预期进球)** 和 **xA (预期助攻)** 贡献占比。
    *   新增工具 `calculate_roster_impact(team_id, missing_players)`。
    *   **逻辑**：如果曼城德布劳内伤停（占全队 xA 25%），工具返回衰减系数 `0.75`。大模型接收后，自动将曼城本场的进攻基数乘以 0.75，再进行后续的所有计算。

### 4. 聪明资金与赔率异动追踪 (Smart Money Tracking)
*   **目标**：捕捉庄家内幕和市场大资金的动向，作为终极防线。
*   **实现方式**：
    *   在后台增加一个轻量级的异步监控脚本 `market_monitor.py`。
    *   监控必发 (Pinnacle) 或马会的核心赔率。如果某场比赛在无重大新闻时，亚盘让球或胜平负赔率发生剧烈震荡（如 1 小时内主胜从 2.1 暴跌至 1.8）。
    *   **动作**：触发 `SmartMoneyAlert` 广播给 `RiskManager`。风控师会立刻拉响警报：“检测到主队有聪明资金异常介入，否决所有客队相关的串关方案，建议单关搏主胜。”

## 阶段三：极致闭环 - 自动寻优与自我进化 (Auto-Tuning)

### 5. 基于反思的参数动态寻优 (Agentic Auto-Tuning)
*   **目标**：让系统每天自动复盘，自我修正凯利准则参数和泊松分布的 Dixon-Coles 修正因子 ($\rho$)。
*   **实现方式**：
    *   摒弃传统的超参数网格搜索。利用系统的 `reflector.py` 机制。
    *   每天凌晨，系统拉取前一天的赛果与自己的预测进行对比。
    *   **LLM 自我反思**：“我昨天在日职联的预测中，过度高估了主场优势（导致 3 场主胜全黑）。我需要调整对日职联的评估权重。”
    *   **动作**：LLM 自动生成补丁，修改本地配置文件 `league_weights.json`，将日职联的主场权重因子从 1.2 降至 1.05。系统完成了基于自然语言逻辑的自我进化。
