# RLEF、GWM 与 ZSA 三合一架构设计规范 (Design Spec)

## 1. 核心架构决策：模式二（双轨并行，长短线分离的“快慢思考”架构）
经过对金融量化套利特性的深度评估，系统将采用**双轨并行（模式二）**的交互范式，辅以**熵驱动（模式三）**的动态路由。

**选择原因**：
在体育博彩中，**时间就是金钱**。零样本套利（ZSA）的核心优势是毫秒级的新闻/赔率截胡。如果像模式一那样强制要求 ZSA 先经过世界模型（GWM）长达数秒的沙盘推演才能出票，套利窗口早就被庄家关上了。因此，必须将“快思考”（ZSA）与“慢思考”（GWM）物理分离，统一由 RLEF 进行赛后奖惩反馈。

## 2. 核心组件定义

### 2.1 RLEF 中枢 (Reinforcement Learning from Environment Feedback)
* **职责**：系统的“大脑与海马体”。不直接参与单场比赛的实时计算，而是作为环境反馈的接收器和参数的调度器。
* **输入**：真实环境的 PnL（盈亏）、资金曲线回撤率、出票滑点。
* **输出**：更新 GWM 内部各维度的战术权重（如“防守反击”的胜率加成），调整 ZSA 监听器的触发敏感度。

### 2.2 GWM 慢轨 (Generative World Models)
* **职责**：系统的“战略推演沙盘”。在赛前（Pre-match）和中场休息（Half-time）等时间充裕的节点，基于时空图神经网络（ST-GNN）生成比赛战术走向。
* **特征**：计算重（高消耗）、胜率极度精确、负责寻找长线的“结构性价值（Structural Edge）”。

### 2.3 ZSA 快轨 (Zero-Shot Cross-Market Arbitrage)
* **职责**：系统的“神经反射弧”。实时挂载在内存总线上，监听 Twitter/RSS 与多家博彩公司的 WebSocket。
* **特征**：计算极轻（零样本 LLM 情感分类/正则匹配）、延迟极低（<500ms），负责在“核心受伤”、“极端红牌”等突发事件导致庄家未及调盘前，直接越过 GWM 强行打单。

## 3. 数据流与控制流 (Data Flow)

1. **环境监听 (Perception)**：
   * 比赛基础面数据（赛程、历史）流向 GWM。
   * 毫秒级突发事件（新闻、赔率秒降）流向 ZSA。
2. **决策执行 (Action)**：
   * GWM 通过慢思考产出 `Expected Value > 1.05` 的稳健注单，送入路由。
   * ZSA 一旦触发阈值，直接无视 GWM 的当前结论，生成“紧急截胡”注单，并带有最高执行优先级。
3. **闭环反馈 (Reward & Backprop)**：
   * 无论哪条轨道出票，赛后其 PnL 都会流入 RLEF。
   * RLEF 如果发现 ZSA 经常因为“假新闻”亏损，会自动下调 ZSA 的新闻信任权重；如果发现 GWM 经常在“雨天”推演错误，会强制修改 GWM 对天气因子的惩罚参数。

## 4. 错误处理与防灾 (Error Handling & Fallbacks)
* **ZSA 假阳性防护**：ZSA 的单笔投注上限被 RLEF 严格锁死（例如最大仓位的 2%），防止被黑客散布的虚假伤病推文“爆仓”。
* **GWM 算力降级**：当同时有 30 场比赛开打时，GWM 如果算力触达瓶颈，将自动退化为传统的泊松分布计算（Fallback to Hardcore Quant Math），保障系统不宕机。

## 5. 阶段性实施计划 (Phasing)
* **Phase 1 (ZSA 快轨)**：优先实现社交新闻监听器与跨市场（如竞彩 vs 平博）的快速价差捕捉器，完成套利闭环，验证“快思考”能力。
* **Phase 2 (RLEF 闭环)**：接入真实盈亏数据，改造 `auto_tuner_agent.py`，让系统能根据快轨的亏损自动调节其触发阈值。
* **Phase 3 (GWM 慢轨)**：引入 ST-GNN 模块或高级图谱分析，替换掉现有的纯泊松分布模型，完成终极进化。
