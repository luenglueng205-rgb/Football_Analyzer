# Time-Machine Backtester & Self-Evolution Engine Plan

## Goal
Ignite the "Digital Life" by activating the Ultimate Backtesting Sandbox and Auto-Tuning Reflection Engine. The system will consume the 200k+ historical matches in the background, run its current predictive models against past realities, and automatically adjust its internal hyperparameters if it fails.

## Core Components to Build/Activate

### 1. The Time-Machine Sandbox (`standalone_workspace/tools/backtest_sandbox.py`)
- **Purpose:** A virtual environment that simulates past dates.
- **Mechanism:** 
  - Takes a historical date (e.g., "2018-06-15").
  - Hides all future knowledge from the Agent.
  - Feeds the pre-match odds to the `SyndicateOS` and records the AI's betting decisions.
  - Compares the decisions against the actual historical match results.
  - Calculates Virtual PnL (Profit and Loss).

### 2. The Auto-Tuner & Reflection Engine (`standalone_workspace/agents/auto_tuner_agent.py`)
- **Purpose:** The self-awareness module.
- **Mechanism:**
  - Analyzes the Virtual PnL report generated by the Sandbox.
  - Uses an LLM to "reflect" on *why* a bet lost (e.g., "I trusted the Home Win too much on high xG variance").
  - Dynamically rewrites the weights in `configs/hyperparams.json` (e.g., increases the `contrarian_weight` or `poisson_variance_penalty`).

### 3. Hyperparameter Configuration (`standalone_workspace/configs/hyperparams.json`)
- **Purpose:** The genetic code of the AI.
- **Mechanism:** Stores the weights for Fundamental, Contrarian, Smart Money, and Poisson scaling factors.

### Execution Steps
1. Create `configs/hyperparams.json` with initial default weights.
2. Implement `backtest_sandbox.py` to run historical simulations.
3. Upgrade `auto_tuner_agent.py` to read sandbox results and write to `hyperparams.json`.
4. Create a test script `scripts/run_evolution_cycle.py` to demonstrate one full cycle of "Predict -> Fail -> Reflect -> Evolve".
